How to contribute to the Ethereum Cypherpunk Research Project

üìù Making Pull Requests

We welcome contributions! Here's how to submit a pull request:

1. Fork & Clone

git clone https://github.com/YOUR-USERNAME/web3-privacy-ethereum-cypherpunk-research.git
cd web3-privacy-ethereum-cypherpunk-research

2. Create a Branch

git checkout -b feature/your-feature-name

3. Make Your Changes

  • Add new project data to deliverables/[project-name]/
  • Update documentation if needed
  • Follow the existing file structure and naming conventions
  • Ensure all data follows our constitutional requirements (100% real data, no fabrication)

4. Commit & Push

git add .
git commit -m "Add [project-name] research data"
git push origin feature/your-feature-name

5. Open Pull Request

Go to GitHub and click "New Pull Request". Our team will review your contribution!

üéØ Types of Contributions

  • New Project Data: Research and add privacy projects we're missing
  • Data Corrections: Fix inaccurate or outdated information
  • Documentation: Improve guides, README files, or explanations
  • Visualizations: Create new data visualizations or charts
  • Code Improvements: Enhance search, UI, or backend functionality
  • Bug Fixes: Identify and fix issues in the codebase

‚úÖ Data Quality Standards

All contributions must follow these rules:

  • Real Data Only: No synthetic or fabricated information
  • Source Verification: Provide sources for all claims
  • Multi-source Confirmation: Critical facts need 2+ sources
  • Confidence Scoring: Rate data confidence 0.0-1.0
  • Report Gaps: If data is missing, report it - don't make it up

See our Constitution (v2.0.0) for complete details.
